1.금융ERP 구축 프로젝트 수행(SecureDB)
고객사인 삼성화재와 삼성생명이 함께 진행한 금융ERP 프로젝트에 SAP DB암호화 솔루션 도입 역할로 참여하여 프로젝트를 수행하였습니다.
당시 제 역할은 PL역할로 참여하였고, 프로젝트 수행과 레거시 암호화 솔루션 운영역할을 함께 진행하였습니다.
프로젝트에서의 주 업무는 ERP전환 시스템을 대상으로 데이터 마이그레이션(레거시 암호화 데이터를 평문 변환 후 SAP DB암호화 솔루션으로 재 암호화)
작업을 수행하였고 오픈이후 레거시와 SAP DB암호화 솔루션을 운영하였습니다.
당시 사용된 기술은 SAP DB암호화 솔루션(SecureDB)과 일부 시스템에서 인터페이스 작업 시 필요한 JAVA, 그리고 DB는 DB2를 사용하였고 미들웨어는 아파치/톰캣을 이용하였습니다.

2. 삼성화재 개인정보 암호화 솔루션 (D'Amo, SafeDB, CubeOne)
고객사인 삼성화재의 레거시 암호화 솔루션을 운영하였습니다.
솔루션은 3가지로 D'Amo와 SafeDB, CubeOne 시스템을 운영하였고 차후에는 CubeOne솔루션을 걷어내면서 D'Amo를 통해 다시 암호화 하는 마이그레이션 작업을 수행하였습니다.
SAP DB암호화 솔루션 이전에는 삼성화재의 전체 개인정보 암호화를 D'Amo와 SafeDB를 통해 진행하였고, 월별로 신청을 받아 신규 시스템 암호화 Agent 설치 및 적용 그리고 신규정책 배포작업을 수행하였습니다.
그 외 역할로는 월 1회이상 개인정보 암호화 미처리 데이터를 점검하여 조치계획을 수립하고 각 담당자로 하여금 조치일정을 조치완료여부에 대해 취합하여 월별 레포트를 작성하여 보고하였고
이 자료를 기반으로 감사대응(국세청, 금감원, ISMS 등) 역할을 수행하였습니다.
당시 사용된 기술은 암호화 솔루션(D'Amo, SafeDB, CubeOne), JAVA, DB2, Oracle, 미들웨어는 아파치/톰캣을 이용하였습니다.
구축된 서버는 Linux, Unix, Windows(NT) 서버등에 구축되어 각종 서버를 접해보았습니다.

3. 데이터 변환/전송 솔루션 (SQLCanvas)
금융사의 경우 망분리작업이 철저하게 지켜지기때문에 운영의 데이터를 개발계로 이관하여 테스트용으로 사용할 수 없었습니다.
하여 고객사에서는 SQLCanvas Trans라는 솔루션을 도입하였는데 이 솔루션은 운영계의 데이터(개인정보 포함)를 개발계로 정해진 건수 또는 전체데이터를 벌크로 이관할 수 있는 솔루션이었고
개인정보의 경우 각 솔루션의 암호화키를 활용하여 운영계 키로 암호화 된 개인정보를 복호화 하고 걔발계로 이관 후 정해진 규칙에 따라 알 수 없는 값으로 변환작업하여 개발계 키로 암호화 하는 작업을 수행하였습니다.
SAP DB암호화 솔루션 적용 후 월 1회이상 테스트 데이터 생성 작업을 수행하였는데 운영계의 데이터를 개발계로 SQLCanvas를 통해서 마이그레이션 하는 작업도 수행하였습니다.

4. 클라우드 EDI 솔루션 구축 프로젝트
신세계아이앤씨 자체 솔루션 구축프로젝트로 유통사와 고객사(제조사)간 전자문서 교환 시스템을 SaaS형태로 구축하였습니다.
당시 역할은 PL로써 솔루션의 인프라 아키텍처를 AWS에 구성하였습니다.
업무별로 아키텍처 구성방식이 달랐는데 웹서비스의 경우 일반적인 인프라 구조(서버 이중화, L4(load balancer)) 로 아키텍처를 구성하였고
특정업무인 웹스크래핑 서비스의 경우 컨테이너 방식을 이용하는 도커, 쿠버네티스 방식을 통해 수집대상 문서별로 컨테이너를 띄워 유통사의 데이터를 수집하는 아키텍처를 구성하였습니다.
형상관리 및 배포의 경우 gitlab과 jenkins를 구성하여 소스형상관리와 배포 및 테스트 자동화 방식을 구축하였습니다.
이번 프로젝트의 경우 특이하게 AA나 TA가 없이 수행되었기에 직접 아키텍처를 구성하고 필요 SW를 설치하는 작업을 진행하였습니다.
오픈이후 시스템운영과 인프라 운영을 담당하였습니다. 
현재는 고도화 프로젝트인 REST API서비스를 구축프로젝트 수행 후 시스템을 오픈하였고 현재 1개 제휴사를 통해 서비스를 제공하고 있습니다.
주로 사용된 기술로는 Java, RDS(MariaDB), React, 서버는 EC2 Amazon Linux, Windows(NT) 미들웨어로는 아파치/톰캣과 스프링부트를 이용하였습니다.
