# Self-Hosted Web IDE Platform

Docker 기반의 Self-Hosted Web IDE 플랫폼입니다. Eclipse Theia를 사용하여 동적으로 개발 환경을 생성할 수 있습니다.

## 프로젝트 구조

```
my-cloud-ide/
├── images/
│   └── java-ide/
│       ├── Dockerfile          # Java 개발 환경 Docker 이미지
│       └── entrypoint.sh       # Git clone/pull 및 Theia 실행 스크립트
├── traefik/
│   └── docker-compose.yml      # Traefik 리버스 프록시 설정
├── scripts/
│   └── launch-ide.sh           # IDE 컨테이너 실행 스크립트
└── README.md
```

## 사전 요구사항

- Docker Desktop (M1 Mac 호환)
- Git

> **⚠️ Docker가 설치되어 있지 않다면?**  
> `DOCKER_SETUP.md` 파일을 참고하여 Docker Desktop을 먼저 설치해주세요.

## 설치 및 실행 순서

> **참고**: 모든 명령어는 프로젝트 루트 디렉토리(`my-cloud-ide`)에서 시작하거나, 각 단계에서 프로젝트 루트로 돌아가서 실행하세요.

### 1. Java IDE Docker 이미지 빌드

```bash
# 프로젝트 루트에서 실행
cd images/java-ide
docker build -t java-ide:latest .
cd ../..  # 프로젝트 루트로 돌아가기
```

또는 프로젝트 루트에서:

```bash
docker build -t java-ide:latest ./images/java-ide
```

### 2. Traefik 시작

```bash
# 프로젝트 루트에서 실행
cd traefik
docker-compose up -d
cd ..  # 프로젝트 루트로 돌아가기
```

또는 프로젝트 루트에서:

```bash
cd traefik && docker-compose up -d && cd ..
```

**Traefik Dashboard 접속:**
- Dashboard: http://localhost:8080/dashboard/
- API: http://localhost:8080/api/rawdata

> **참고**: Dashboard는 `/dashboard/` 경로로 접근해야 합니다 (마지막 슬래시 포함).

### 3. IDE 컨테이너 실행

```bash
# 프로젝트 루트에서 실행
cd scripts
chmod +x launch-ide.sh
./launch-ide.sh [프로젝트명] [GIT_URL (선택사항)]
cd ..  # 프로젝트 루트로 돌아가기
```

또는 프로젝트 루트에서:

```bash
chmod +x scripts/launch-ide.sh
./scripts/launch-ide.sh [프로젝트명] [GIT_URL (선택사항)]
```

**예시:**

Git 저장소와 함께 시작:
```bash
./scripts/launch-ide.sh my-java-project https://github.com/spring-projects/spring-boot.git
```

Git 저장소 없이 빈 프로젝트로 시작:
```bash
./scripts/launch-ide.sh my-java-project
```

> **참고**: GIT_URL을 제공하지 않으면 빈 프로젝트 디렉토리로 IDE가 시작됩니다. 나중에 IDE 내에서 Git 저장소를 클론하거나 새 프로젝트를 생성할 수 있습니다.

### 4. IDE 접속

브라우저에서 다음 URL로 접속:
```
http://[프로젝트명].localhost
```

예시:
```
http://my-java-project.localhost
```

## 주요 기능

### Java IDE Docker Image

- Base Image: `node:18-bullseye` (M1 호환)
- Theia CLI를 사용하여 Eclipse Theia 실행
- 설치된 패키지:
  - OpenJDK 17
  - Git
  - cURL
- 자동 Git Clone/Pull: `GIT_URL` 환경변수가 설정되면 자동으로 저장소를 클론하거나 업데이트합니다.

### Traefik 설정

- 포트 80에서 리버스 프록시 실행
- Docker Socket 마운트로 동적 라우팅 지원
- Dashboard 활성화 (포트 8080)

### 실행 스크립트

- 자동으로 Traefik 네트워크에 연결
- 동적 호스트 라우팅 설정
- 컨테이너 중복 실행 방지

## 유용한 명령어

### 컨테이너 로그 확인
```bash
docker logs -f [프로젝트명]-ide
```

### 실행 중인 IDE 컨테이너 목록
```bash
docker ps --filter "name=-ide"
```

### 컨테이너 중지
```bash
docker stop [프로젝트명]-ide
```

### 컨테이너 삭제
```bash
docker rm -f [프로젝트명]-ide
```

### Traefik 중지
```bash
# 프로젝트 루트에서 실행
cd traefik && docker-compose down && cd ..
```

또는:

```bash
docker-compose -f traefik/docker-compose.yml down
```

## 문제 해결

### Docker 명령어를 찾을 수 없는 경우

Docker Desktop이 설치되어 있지 않은 상태입니다. `DOCKER_SETUP.md` 파일을 참고하여 설치하세요.

### "failed to connect to the docker API" 오류

Docker Desktop이 **실행되지 않은 상태**에서 발생하는 오류입니다.

**해결 방법:**
1. Applications에서 **Docker.app 실행**
2. 메뉴바에 Docker 아이콘(고래 모양)이 나타날 때까지 대기
3. 터미널에서 `docker ps` 명령어로 확인

자세한 내용은 `DOCKER_SETUP.md` 파일을 참고하세요.

### "pull access denied" 또는 "repository does not exist" 오류

`theiaide/theia:next` 이미지가 존재하지 않거나 접근할 수 없는 경우 발생하는 오류입니다.

**해결 방법:**
- 이미 Dockerfile이 수정되어 `node:18-bullseye` 기반으로 변경되었습니다.
- Theia CLI를 사용하여 직접 실행하도록 구성되어 있습니다.
- 다시 빌드해보세요:

```bash
cd images/java-ide
docker build -t java-ide:latest .
```

### M1 Mac 호환성

현재 Dockerfile은 `node:18-bullseye` 기반으로 M1 Mac과 호환됩니다. 만약 빌드 오류가 발생하면:

```bash
docker build --platform linux/amd64 -t java-ide:latest .
```

### 포트 충돌

포트 80이 이미 사용 중인 경우, `traefik/docker-compose.yml`에서 포트를 변경할 수 있습니다:

```yaml
ports:
  - "8080:80"  # 예시: 8080으로 변경
```

그리고 `launch-ide.sh`에서 접속 URL도 변경해야 합니다.

### Git Clone 실패

- Git URL이 올바른지 확인
- Private 저장소의 경우 SSH 키 설정이 필요할 수 있습니다 (향후 개선 예정)
