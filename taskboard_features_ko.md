# 테스크보드(TaskBoard) 기능 명세서

현재 구현된 TaskBoard 어플리케이션의 주요 기능을 사용자 역할(관리자/일반 사용자)에 따라 정리한 내용입니다.

## 1. 계정 및 시스템 (공통)
모든 사용자가 이용 가능한 기본 시스템 기능입니다.
- **인증 시스템**: 아이디/비밀번호 기반 로그인 및 세션 유지.
- **환경 설정**:
    - **다국어 지원**: 한국어(KO) 및 영어(EN) 전환 가능.
    - **테마 설정**: 라이트 모드, 다크 모드, 시스템 설정 따르기 지원.
    - **반응형 디자인**: 데스크탑 및 모바일 환경에 최적화된 레이아웃.

## 2. 대시보드 (프로젝트 목록)
| 기능 | 관리자 (Administrator) | 일반 사용자 (General User) |
| :--- | :--- | :--- |
| **프로젝트 조회** | **모든 프로젝트** 조회 가능 (권한 무관). | 본인이 **소유**하거나, **멤버**로 참여 중이거나, **태스크가 할당**된 프로젝트만 조회. |
| **프로젝트 생성** | 새로운 프로젝트 생성 가능. | 새로운 프로젝트 생성 가능 (자동으로 **프로젝트 소유자**가 됨). |

## 3. 프로젝트 관리 (칸반 보드)
| 기능 | 관리자 (Administrator) | 일반 사용자 (General User) |
| :--- | :--- | :--- |
| **프로젝트 수정** | **모든 프로젝트**의 정보(제목, 설명, 기간) 수정 가능. | **본인이 소유한** 프로젝트만 수정 가능. |
| **프로젝트 삭제** | **모든 프로젝트** 삭제 가능. | **본인이 소유한** 프로젝트만 삭제 가능. |
| **리스트(컬럼) 관리** | 모든 프로젝트의 리스트 생성, 이름 변경, 삭제 가능. | 접근 가능한 프로젝트의 리스트 생성, 이름 변경, 삭제 가능. |
| **태스크 관리** | 모든 프로젝트의 태스크 생성, 수정, 삭제, 이동 가능. | 접근 가능한 프로젝트의 태스크 생성, 수정, 삭제, 이동 가능. |

## 4. 태스크 보드 상세 기능
- **동적 워크플로우 (Dynamic Workflows)**
    - 사용자가 직접 리스트(컬럼)를 추가하여 원하는 작업 흐름(예: 할 일 -> 진행 중 -> 검토 -> 완료)을 구성할 수 있습니다.
- **드래그 앤 드롭 (Drag & Drop)**
    - **순서 변경**: 같은 리스트 내에서 태스크 순서를 자유롭게 변경.
    - **상태 변경**: 태스크를 다른 리스트로 이동하여 진행 상태 업데이트.
    - **데이터 영속성**: 이동한 위치와 리스트 정보는 데이터베이스에 즉시 저장되어, 로그아웃 후에도 그대로 유지됩니다.
- **태스크 상세 정보**
    - **기본 관리**: 태스크 생성(Create), 조회(Read), 수정(Update), 삭제(Delete).
    - **속성**: 제목, 상세 설명, 마감일(Deadline), 태그(General, Urgent, Bug, Feature 등).
    - **담당자 지정**: 프로젝트 멤버 중 담당자를 다중 선택하여 할당 가능.

## 5. 권한 요약
- **관리자 (Administrator)**: 시스템의 '슈퍼 유저'로서 모든 데이터에 접근하고 수정/삭제할 수 있는 권한을 가집니다. 운영 및 관리에 적합합니다.
- **프로젝트 소유자 (Owner)**: 본인이 생성한 프로젝트에 대해 완전한 제어 권한을 가집니다.
- **멤버 (Member)**: 초대받거나 할당된 프로젝트 내에서 태스크와 리스트를 자유롭게 관리할 수 있습니다.
